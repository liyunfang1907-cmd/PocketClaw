## PocketClaw 简介
PocketClaw 是基于 OpenClaw 开源项目的便携式个人 AI 助手，部署在 U 盘（ExFAT 格式）中，随插随用。通过 Docker 容器运行，默认端口 18789。默认使用智谱 GLM-4.7-Flash 免费模型（200K 上下文）。

## 目录结构（U 盘中 PocketClaw 文件夹下）
- docker-compose.yml — 容器编排配置
- Dockerfile.custom — 自定义镜像构建文件
- VERSION — 版本号文件
- PocketClaw.bat — Windows 统一启动器（主菜单）
- PocketClaw.command — macOS 统一启动器
- config/openclaw.json — PocketClaw 主配置文件（模型、Agent 等核心设置）
- config/providers.json — 模型提供商配置（API 地址、模型列表）
- config/setup.html — Web 配置面板页面
- config/mobile.html — 移动端界面
- config/workspace/ — AI 系统配置（AGENTS.md、SOUL.md、skills/ 等），容器内映射为 workspace
- 工作区/ — 用户文件夹（AI 创建的文档、用户放入的文件），容器内映射为 workspace/工作区
- data/credentials/ — 频道凭证（加密存储）
- data/sessions/ — 会话历史数据
- data/logs/ — 运行日志（含 doctor 诊断报告）
- secrets/.env.encrypted — AES-256-CBC 加密的环境变量（含 API Key）
- scripts/ — 管理脚本目录

## 常用脚本（scripts/ 目录下，.bat 为 Windows，.sh 为 macOS/Linux）
- start.bat / start.sh — 启动服务
- stop.bat / stop.sh — 停止服务
- setup-env.bat / setup-env.sh — 首次运行配置向导（设置 API Key 等）
- change-api.bat / change-api.sh — 修改 API Key
- encrypt.bat / encrypt-secrets.sh — 加密 .env 文件
- decrypt.bat / decrypt-secrets.sh — 解密 .env 文件
- backup.bat / backup.sh — 备份数据
- update.bat — 更新 PocketClaw
- doctor.bat / doctor.sh — 系统诊断修复
- logs.bat — 查看容器日志
- status.bat — 查看容器状态
- reset.bat / reset.sh — 重置服务

## 日常使用流程
- 启动：插入 U 盘 → 双击 PocketClaw.bat → 选 [1] 启动 → 输入 Master Password → 等待容器启动
- 聊天：浏览器打开 http://127.0.0.1:18789/chat#token=<启动时显示的Token>
- 停止：PocketClaw.bat → 选 [2]，或双击 scripts/stop.bat
- 首次使用需运行 setup-env 配置向导设置 API Key，然后 encrypt 加密保护

## 模型配置
- 默认模型：智谱 GLM-4.7-Flash（永久免费）
- API 地址：https://open.bigmodel.cn/api/paas/v4
- 申请 Key：https://bigmodel.cn
- 配置文件：config/providers.json（模型列表）、config/openclaw.json（默认模型选择）
- 支持切换其他 OpenAI 兼容 API（硅基流动、DeepSeek 等）

## Doctor 工具箱
doctor.bat 除了自动诊断，还提供以下工具箱操作：
- [R] 刷新 Token：停止并重启容器，生成新的 Gateway Token（解决 token mismatch 问题）
- [B] 重建镜像：执行 docker-compose build --no-cache，重新构建 PocketClaw 镜像（解决镜像损坏或需要更新自定义镜像）
- [A] 自动修复：自动执行常规修复（重置容器、重建镜像等）
- [S] 跳过：跳过修复操作

## 关于 token mismatch
日志中出现 "gateway token mismatch" 或 "unauthorized" 通常表示浏览器缓存了旧的 Token。解决方法：
1. 关闭所有浏览器标签页
2. 运行 doctor 工具箱的 [R] 刷新 Token
3. 用新 Token URL 重新打开浏览器

## 常见问题
- Docker 未安装：首次启动 PocketClaw.bat 会自动引导安装
- WSL2 问题：首次安装 WSL2 后需重启电脑一次
- 端口冲突：默认 18789，可在 docker-compose.yml 中修改
- 容器无法启动：检查 Docker Desktop 是否运行，或用 doctor 诊断
- API Key 无效：用 change-api.bat 重新设置，或在 Web 面板修改
